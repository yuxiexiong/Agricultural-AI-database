# TECH-001：农业 LLM 的正确架构


素材类型：技术架构  
推荐架构：

```text
用户问题/任务
→ 意图识别
→ 地区/作物/生育期/用户身份补全
→ RAG 检索：农技知识库 + 政策 + 本地 SOP + 历史案例
→ 工具调用：天气、地块、传感器、遥感、作业记录
→ 规则校验：农药/兽药/安全/合规/禁用项
→ 生成建议：解释 + 依据 + 置信度 + 风险提示
→ 专家审核/用户反馈
→ 日志与知识库更新
```

核心观点：农业 LLM 不是一个聊天框，而是“农业知识、实时数据、规则约束和专家流程”的自然语言入口。
